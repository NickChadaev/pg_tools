Валенков.
---------
В оригинальную БД CRM пользователям Metabase доступ обеспечивать не требуется, 
они не должны иметь доступа ни к её структуре, ни к данным в ней. 
Для BI-задач региональных компаний должен быть развернут отдельный postgres-кластер 
(с одной БД с предоставлением доступа РГК по отдельным схемам, либо с отдельной БД для каждой РГК 
(в этом кластере). 

Доступ групп пользователей Metabase нужно ограничивать именно этими категориями, 
для каждой группы должно быть создано отдельное подключение в БД, 
обеспечивающее доступ только к данным соответствующей РГК (на уровне БД через роли postgres). 
В каждой из схем/БД РГК будет создана согласованная позднее структура таблиц, которая будет 
с определенной регулярностью заполняться из оригинальной БД CRM. 
Всё необходимые отчеты и дашборды пользователи РГК будут стоить на таких выгружаемых данных

Run Metabase with Docker and Docker Compose, using PostgreSQL as application database.

Based on official docker images for PostgreSQL and Metabase.
Usage

    Clone this repository.

    Copy .env.sample to .env and adjust the variables.

    Start services locally using Docker Compose.

    $ docker compose up

Metabase can connect to host (host.docker.internal) from static IP 172.16.200.30, (static IP can be used for authentication. e.g. in pg_hba.conf)
References

Get started

    Run docker-compose build to create a database image (mysql or postgres)
    Run docker-compose up to run the applications
    Use docker-compose down to shutdown all services
    Use docker-compose up to re-run all services whenever you want
    There is no need to re-run docker-compose build unless you change the database strategy, database name, database credentials or the volumes.
