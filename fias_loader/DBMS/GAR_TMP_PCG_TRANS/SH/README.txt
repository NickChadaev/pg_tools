0) Убедиться в том, что клиент Postgres psql доступен к командной строке.
   Для это необходимо набрать команду psql без параметров. Если он ругается,
   значит всё хорошо. Делов том, что используемый в процессе обновления скрипт
   load_upd.sh, является обёрткой над psql.

1) Проверит правильность настроек .pgpass. В противном случае предстоит
   255 раз ввести пароль пользователя postgres.    

2) Скачать архив up_prds_YYYMMDD.tar.gz, содержащий инкрементальные обновления.

2) Выполнить команду tar -xvzf up_prds_YYYMMDD.tar.gz. 
   После распаковки архива появится каталог up_prds_YYYMMDD, далее перейти в него.
  
3) Выполнить скрипт load_upd.sh. Если он вызвается без параметров,
   то выдаст подсказку по использованию, например:
   $>./load_upd.sh            
   Использование: load_upd.sh <IP хоста> <Порт> <Пользователь> <Имя базы>

   2023-01-23 Добавлен новый скрипт load_upd_wi.sh - который выполняет обновление
   ----------  без перестройки индексного покрытия.    

4) Выполнить обновление базы   
   $>./load_upd.sh 127.0.0.1 5432 postgres <имя БД>
          либо 
   $>./load_upd_wi.sh 127.0.0.1 5432 postgres <имя БД>
   
5) ОБЯЗАТЕЛЬНО вернуть мне два LOG-файла: process.out и process.err.
